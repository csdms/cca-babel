Bindings generation

Much of the information for generating client-side bindings is pertinent to implementing a SIDL class in C++. SIDL type mappings are listed in Table 7.1. An implementation can call other SIDL methods, in which case the rules for client calls must be followed.

To create the implementation, a valid SIDL file must be generated by invoking Babel as follows:

% babel -exclude-external -server=UC++ file.sidl
or simply
% babel -E -sUC++ file.sidl

As a result, a makefile fragment called babel.make, several C header and source files, and numerous C++ header and source files are created. The only files that need to be hand-edited are the C++ ``Impl'' files (i.e., header and source files that end in _Impl.hxx or _Impl.cxx). More on this in Subsection 7.4.2.
